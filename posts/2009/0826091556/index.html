マイデスクトップその2「開発用サーバ、ls ~」編

Posted at — Aug 26, 2009

本来なら机の次は、メインのクライアントマシン Mac mini や ディスプレイを扱おうと思ったんですが、 TAKESAKO さん曰く「ls ~」を晒すのが流行っているらしい ので、 「マイ ls ~ 」を晒しつつ、開発用として使っているサーバの紹介します。

サーバと言ってもこれも昨日紹介した机に載っています。 Shuttle の X27D という 4コアな Atom CPU のべアボーンに SSD 入れたものです。 写真にあるポカリの中身が透明なのは気にしないでください。

なんで Mac あるのにそっちで開発しないの？って感じですが、 どうも debian etch が好きなのか、本番環境と揃えてテストしたいから、いろいろ自由に実験したいからか、 なんとなく Mac mini の Terminal からいつもこのマシンにログインして開発してます。 そしてこの開発機のホームディレクトリはさらに納戸にあるファイルサーバのNFSをマウントしているという具合です。 べアボーン自体は2万円程度で手に入り、かつ消費電力もそんな高い、性能もちょっと emacs の立ち上がりがもっさりかなって ぐらいで慣れれば気にならない。そしてもちろん、机に載るほど小さいという点でお気に入りです。

さて、ではホームディレクトリで ls してみましょう。 

 yusuke@dev:~$ ls
Desktop  changelogmemo  lab                public_html  src   wadit
Maildir  documents      lib                repos        temp  work
bin      groups         memcache-top-v0.6  shipit-dist  text  www

memcache-top がなぜか直下にあるくらいで、特に面白い点はないですね。 というのも、開発中のコードとかは全て ~/work 以下にディレクトリを掘って置いてあります。 では、その work ディレクトリを、なんかヤバそうなのがありそうだけど、 それは無視する方向で公開してみましょう。
